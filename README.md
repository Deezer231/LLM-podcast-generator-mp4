# LLM Podcast Generator

## Overview
A full-stack web app that generates AI (Used a LLM) -powered podcasts from a topic prompt, stores them, and allows users to register, login, and listen/download generated podcasts.

Figma Wireframe Link: https://www.figma.com/design/w8SSCzBFSoHZctkjLxSAqE/LLM-Podcast-Generator-wireframe?node-id=1-3&t=4ENstLJ3vkcJQOdC-1
## Features
- User registration and login (MySQL backend, passwords hashed with bcrypt)
- Podcast generation using GPT4All Falcon (runs locally/offline)
- Audio merging and download
- MongoDB for podcast storage
- Protected dashboard and podcast list for logged-in users

## How It Works!


### Chunk-based Google Translate TTS (Text-to-Speech)
- The podcast script generated by the AI can be very long, but Google Translate's TTS API only allows short text segments.
- The backend splits the script into ~200-character chunks.
- Each chunk is sent to Google Translate's TTS endpoint to generate a short MP3 audio file.
- All audio chunks are downloaded and saved locally.
- The backend then merges all the audio chunks into a single MP3 file using ffmpeg.
- The final audio file is served to the frontend for playback and download.

### Step-by-Step: What Happens When You Generate a Podcast
1. **User enters a topic and clicks 'Generate Podcast' on the frontend.**
2. **Frontend sends a POST request to the backend `/generate` endpoint with the topic.**
3. **Backend process:**
   - Calls GPT4All Falcon to generate a podcast script about the topic (instructed to avoid hallucination and roleplay).
   - Calls GPT4All Falcon again to generate a title and 3 bullet points about the topic.
   - Splits the script into ~200-character chunks.
   - For each chunk, downloads TTS audio from Google Translate.
   - Merges all audio chunks into a single MP3 file using ffmpeg.
   - Saves the podcast (title, topic, script, bullet points, audio URL) to MongoDB.
4. **Backend responds to the frontend with the new podcast data and audio URL.**
5. **Frontend displays the generated podcast, script, bullet points, and audio player.**
6. **User can listen to or download the full podcast audio.**

## Tech Stack
- **Frontend:** React (Create React App)
- **Backend:** Node.js, Express
- **Database:** MongoDB (podcasts), MySQL (users + passwords + emails)
- **AI:** GPT4All Falcon (local LLM)

## Prerequisites
- **Node.js** (v18+ recommended)
- **npm** (v8+ recommended)
- **Python 3.x** (required by GPT4All backend)
- **MongoDB** (local or Atlas)
- **MySQL** (local or remote (I used phpmyadmin))

## Setup Instructions

### 1. Clone the repository
```bash
git clone <your-repo-url>
cd podcast generator
```

### 2. Environment Variables
Create a `.env` file in `PD-BACKEND/` (see `.env.example`):
```
MONGODB_URI=your_mongodb_uri
MYSQL_HOST=your_mysql_host
MYSQL_USER=your_mysql_user
MYSQL_PASSWORD=your_mysql_password
MYSQL_DATABASE=your_mysql_db
```

### 3. Install dependencies
```bash
# Backend
cd PD-BACKEND
npm install
npm install connect-mongo

# Frontend
cd ../podcast-frontend
npm install
```

### 4. Download GPT4All Model
- Download the `gpt4all-falcon-newbpe-q4_0.gguf` model from [GPT4All downloads](https://gpt4all.io/index.html) OR (download GPT4ALL -> Models -> install GPT4ALL Falcon(3.92 GB)) and place it in the backend directory (or update the path in `gpt.js`).

### 5. Start the servers

# In one terminal (backend)
cd PD-BACKEND
node server.js

# In another terminal (frontend)
cd podcast-frontend
npm start


### 6. Usage
- Visit `http://localhost:3000` for the guest generator.
- Register/login to access the dashboard and full podcast list.

## Security & AI Disclaimer
- **Never commit your real `.env` file or credentials.**
- **Passwords are securely hashed with bcrypt before storing in MySQL.**
- **AI-generated podcasts may contain false information or hallucinated details. Always fact-check important content.**
